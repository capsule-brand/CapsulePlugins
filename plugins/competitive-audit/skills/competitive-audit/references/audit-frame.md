# Audit Framework: Wins / Parity / Gaps + A/B/C/D Recommendations

The analytical spine of the audit. Every audit produces three sections of comparison findings (§6) and four buckets of recommendations (§7).

## Wins / Parity / Gaps

For §6 of the document. Honest, evidence-based, ordered by salience.

### Wins (where the client clearly leads)

Criteria for inclusion:
- The advantage is defensible (not just "we shipped first")
- It's visible to a buyer in the first 5 minutes of evaluation
- A competitor would need months or significant org change to copy it

Common patterns:
- **Pricing posture** — radically cheaper, simpler model (pay-as-you-go vs. enterprise contract)
- **Multi-segment breadth** — one tool that covers what 3 competitors cover separately
- **AI-native UX** — the flagship interaction is faster than incumbents' workflow-first products
- **Time to value** — minutes-to-first-output vs. days-to-onboard

For each win, include one specific evidence point — a price, a feature pair, a screenshot detail.

### Parity (category table stakes)

Things the client has that every competitor also has. Important to name so the audit doesn't accidentally over-claim.

Common patterns:
- "FDA / EU / SOC 2 / SSO" — depending on category
- "Saves weeks of work" — every vendor uses this line
- "Cloud-based, no install" — meaningless in 2026

The point of listing these: discipline. If a draft is calling these "wins," fix it.

### Gaps (where the client lags)

This is where the recommendations come from. Order by severity (deal-breaker → cosmetic).

Common patterns:
- Trust signals missing (SOC 2, advisors, logos, reviews)
- No enterprise tier (price ladder ends below where enterprise buyers shop)
- Coverage hole (geography, industry, framework)
- Integrations missing (no API, no PIM/DAM/ERP)
- Workflow shallow (no statuses, no approval, no audit log)
- Brand mismatches (footer wrong, copyright wrong, page titles inconsistent)
- Public review profile empty (zero G2/Capterra)

Each gap maps to a recommendation in §7. If a gap doesn't drive a recommendation, either drop the gap or add the recommendation.

## A/B/C/D Recommendation Buckets

For §7 of the document. Ordered by leverage (impact per unit of effort).

### A. Trust & defensibility (do these first / this week)

These are the cheapest, fastest moves that change the buyer's read of the company. Most are website edits.

Standard A-bucket recommendations:
- Fix any brand-mismatch errors (footer copyright, page titles, OG metadata)
- Add a Trust / Security page (SOC 2 status, encryption, data residency — even "in progress" is better than silence)
- Name a regulatory or industry advisor (one ex-FDA / ex-industry name changes credibility)
- Add a customer logo wall and 2–3 testimonials (offer credits in exchange if needed)
- Claim G2 / Capterra / TrustRadius profiles, seed 5–10 reviews
- Lead with the strongest differentiator above the fold (price, breadth, speed)
- Publish a regulation / framework coverage matrix (version-dated, with citation links)

Effort: hours-to-days. Impact: changes the very first impression. Always recommend.

### B. Product features (the 6-month roadmap)

The build-bets that close real gaps. Rank by ROI.

Standard B-bucket recommendations:
- API + integrations (Figma plugin, Adobe Illustrator plugin, REST API, webhooks)
- Bulk / batch features (folder uploads, multi-SKU scanning)
- Approval workflow lite (statuses, comments, audit log)
- Per-tier specialty modules (e.g., a pharma module with regulatory advisor hours)
- Regulation change alerts (drives logins between launches; justifies subscription pricing)
- New geography / framework coverage (Asia, MENA, LATAM in regulated categories)
- Pre-flight claim checker (marketing as a second buyer in the same account)
- Side-by-side version diff with impact analysis

For each, name the buyer segment it serves and the gap it closes.

### C. Pricing & packaging moves

Structural pricing changes. The hardest to undo, so be specific.

Standard C-bucket recommendations:
- Add a Pro Monthly subscription alongside any pay-as-you-go (captures repeat-volume customers)
- Add an Enterprise tier ($25K–$60K/yr range typical) — SOC 2, SSO, dedicated advisor hours, SLAs, integrations
- Separate pharma / regulated / vertical SKUs with distinct messaging and pricing
- Annual prepay discount on the largest non-enterprise tier (15–20%)
- "Try one for free" lead magnet behind a work-email gate
- Referral credits (10 free credits for each referred team that buys)

Avoid generic "raise prices" advice — be specific about which tier and why.

### D. GTM & positioning

Marketing and distribution moves. Higher uncertainty but compounding returns.

Standard D-bucket recommendations:
- Buyer-specific landing pages (one per named persona)
- Headline repositioning to lean on the strongest wedge
- Public benchmark / teardown content (cheap to produce, viral in the category)
- Partnership lane (contract manufacturers, agencies, vertical communities)
- Sponsored content in the 1–2 highest-signal industry newsletters
- Conference presence in the canonical industry event

## Ordering within buckets

Within each bucket, order by leverage. Number them. The user reads the first 2 items of each bucket; the rest sustain a follow-up conversation.

## What good looks like

A good audit's §7 has 4–8 items per bucket, with each item:
- One sentence describing the move
- One sub-bullet on why it matters
- A specific number or detail (price, page name, integration target)

A bad §7 has 12 items per bucket, all generic, no numbers. Don't write that.
